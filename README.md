本题正解：

```c
#define MAX(a, b)   (((a) > (b)) ? (a) : (b))

int MaxSubArray(const int nums[], int numSize)
{
    int sum = 0, res = nums[0];
    for(int i = 0; i < numSize; i++)
    {
        const int n = nums[i];
        sum = MAX(sum + n, n);
        res = MAX(res, sum);
    }
    
    return res;
}

```

## <a name="tube_test_with_mice"></a> 用三只老鼠测八根试管中的一根毒试管的问题

有 **8** 根试管，其中 **只有一根** 试管是有毒的。现在有 **3** 只小白鼠。我们可以通过让小白鼠尝试饮用某一根或某几根试管中混合出来的试剂来测试哪一根试管有毒。现在要求：每只小白鼠只能饮用一次试剂，那如何让这些小白鼠 **同时** 饮用安排好的试剂来 **一次性** 测试出哪根试管有毒？

乍看之下，这个问题似乎比较简单。如果最后不要求 “**同时一次性**” 测试的话，我们最容易想到的是使用折半搜索法来解决。比如，先对8根试管分成前四根与后四根，然后让第一只老鼠先品尝前4根试管混合出来的试剂。如果第一只老鼠没死，那么说明这4根没有毒，而另外4根有毒。随后把推测有毒的4根试管再分成2组，然后让第二只老鼠品尝，前2根试管混合出来的试剂。如果老鼠没死，那说明这两根试剂都没毒，有毒的在另外两根之中；如果死了则说明这两根试剂有其中一根是有毒的。轮到第3只老鼠之后就能判别哪根试剂有毒了——如果死亡，则说明喝到有毒的；若没死，则说明另外一根有毒。

而问题要求是让这3只小白鼠同时品尝试剂，且只能喝一次，一次性得出最终结果。因此我们不能直接用折半搜索算法去解决。而这里有一个显式的关键条件，即——只有一根试剂是有毒的，也就意味着目标对象个数是1。还有一个隐藏的关键条件就是——老鼠品尝试剂后的状态是2值的，“生”或“死”，其中之一。所以我们知道，可以用1只老鼠就能用来测2根试管的毒性。正因为只有1根试管是有毒的，所以对于2根试管而言，只需让1只老鼠品尝其中一根试管，那么凭借它后面是死还是生就能知道哪一根试管是有毒的——如果喝死了，那么喝的那根是有毒的，若是活着，说明另一个是有毒的。

那我们接着尝试用3根试管，看看至少需要几只小白鼠就能测试所有3根试管的毒性。之前我们知道，1只老鼠可以测得2根试管，如果是3根试管的话，1只老鼠显然不够。那我们再加1只老鼠。现在有2只老鼠了。那我们可以让1只老鼠尝试第1根试管，让第2只老鼠尝试第2根试管，这样是否能判定所有3根试管的毒性呢？答案是肯定的。如果第1只老鼠死了，说明第1根试管有毒；如果第2只老鼠死了，那说明第2根试管有毒；如果两只老鼠都没死，那说明第3根试管有毒。

OK，接着我们再尝试4根试管，看看至少需要几只小白鼠就能测试所有这4根试管的毒性。通过刚才测试3根试管的步骤，我们知道2只老鼠完全可以测试3根试管的毒性。那2只老鼠是否能测试4根试管呢？我们这回可以尝试使用“试剂混合”这一特性。倘若把其中两根试管的试剂进行混合，那如果把老鼠喝死了，说明这两根试管中有一根是有毒的；如果没被喝死，则说明这俩试管均无毒。所以现在的问题可以转换为：我们是否能够对这4根试管做“信息压缩”，使得它能用两只小白鼠就能测出有毒的那一根。而将试管进行混合就是一种压缩方式了，因为两根试管混合在一起一次能用于测试2根试管的毒性。这么一来，我们可以减少所要测试的最终样本数（对应于所使用的小白鼠的个数）。我们尝试将第1根试管与第2根试管混合，给第1只老鼠；将第2根试管与第3根试管混合，给第2只老鼠。这么一来能得出一个有意思的结果：如果只有第1只老鼠死了，那说明第1根试管有毒（因为第2根试管的试剂也被第2只老鼠喝了）；如果第2只老鼠死了，说明第3根试管有毒；而如果两只老鼠都死了，则说明第2根试管有毒；而如果两只老鼠都没死，那说明第4根试管有毒（因为前三根试管都没毒而能直接推导出）。

从这一过程来看，我们发现可以把关注点放到小白鼠身上了。为了能最大限度地去压缩试管信息，我们是否能对小白鼠的生死状态来映射为哪一根试管有毒呢？答案是肯定的。正因为老鼠喝了试剂后只有两种状态——生或死，我们不妨把生用 **0** 表示；把死用 **1** 来表示。因为有毒试剂就1根，并且是最终我们所求的目标状态，从而把死亡状态设置为 **1** 将会更为直观。所以，对于3只小白鼠而言，可将其视作为 **3个比特**。于是可获得以下真值表：

bit2 | bit1 | bit0 | number
---- | ---- | ---- | ----
0 | 0 | 0 | 0
0 | 0 | 1 | 1
0 | 1 | 0 | 2
0 | 1 | 1 | 3
1 | 0 | 0 | 4
1 | 0 | 1 | 5
1 | 1 | 0 | 6
1 | 1 | 1 | 7

我们看上表，是不是正好有8个结果~🤭因此，我们现在可以根据上述真值表对8根试管进行编号，依次为0到7。同时对小白鼠也进行编号，依次为0到2。而上述表中 **bit0** 到 **bit2** 的 **1** 则表示对应的试管编号需要给哪些小白鼠品尝。比如，试管3（编码为011）会被老鼠0和老鼠1同时喝到，这么一来，如果这两只老鼠都死了且老鼠2没死，则说明试管3有毒。因此，由此表我们可以得到下表（M表示老鼠，T表示试管，**|** 表示试剂混合）：

- M0（bit0）：T1 | T3 | T5 | T7 （1、3、5、7的比特0均为1）
- M1（bit1）：T2 | T3 | T6 | T7 （2、3、6、7的比特1均为1）
- M2（bit2）：T4 | T5 | T6 | T7 （4、5、6、7的比特2均为1）

布置完之后我们就能得到以下结果：

- M0死（001）：T1有毒
- M1死（010）：T2有毒
- M0与M1一起死（011）：T3有毒
- M2死（100）：T4有毒
- M0与M2一起死（101）：T5有毒
- M1与M2一起死（110）：T6有毒
- 3只老鼠全死（111）：T7有毒
- 3只老鼠全活（000）：T0有毒

整个编码过程完成！
